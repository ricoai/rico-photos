rico.ai Photos

The intent of this project is to upload a image, process the image using AI to determine the semantics of the image, then display all the details of the image. Like an alternative to Google Photos but with more details.
Getting Started

Create an account. Then upload some images. As the image is uploaded, it will be processed and appear on the dashboard. Click on any of the images in the dashboard to see the AI results.

How?
A user logs in and uploads images to the site.

User information is stored in an Microsoft SQL Server hosted on AWS RDS. The SQL Server keeps track of all the AI results and the S3 paths for each image.

The original images and thumbnails are stored in an AWS S3 Bucket. Each user has there own folder.

The image is then processed by AWS Rekognition to determine the semantics of the image.
  • Determine if there are any objects in the images. (Object Detection)
  • Determine if there are any faces and the there facial expressions. (Facial Expressions and Sediment)
  • Determine if there are any sexual content. (Moderation)
  • Determine if there is any text. (Text Detection)
Meta data of the image is obtained to determine information like location where the image was taken, the settings and camera type.

The Frontend uses Angular to upload and display the images. The uploaded images are displayed in a grid of thumbnails. Each thumbnail can be clicked to see all the AI results of the image, including the bounding boxes.

The Backend uses ASP.net MVC to load the SPA site and respond to API calls. It uses EntityFramework to load data to the MS SQL server and maintain the database schema.

The SQL Server keeps track of all the AI results and the S3 paths for each image.
